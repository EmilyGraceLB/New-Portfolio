Balcony Bound

An immersive exploration of unique interactions in a sandbox cosy-style virtual reality environment.

Project Overview

Balcony Bound takes inspiration from EarthBound (Nintendo, 1994) in its title, presenting the feeling of being on track toward a cozy safe haven. This experience takes place on a balcony overlooking a quiet town, where paper planes pass by in the wind. The player is able to explore the balcony and make their own paper planes from the paper stacks on the tables, folding their memories to add to the space.

The Technical Challenges & Solutions

Hurdle 1: Visual Consistency Across Dynamic Lighting

The Problem: Turning off the primary fairy light system caused the cosy balcony environment to look completely flat, pitch black, and visually unappealing in unlit modes.

The Solution: Developed a custom reflective material for the balcony glass doors with its shader emission property mapped directly to the skybox glow. This created a subtle, natural fallback light source that preserved the scene's depth and warmth whether the main lights were toggled on or off.

Hurdle 2: C# Script Architecture

The Problem: Attempting to handle object instantiation, spatial audio injection, and playback timers inside a single script became highly cluttered [1] and prone to performance bugs.

The Solution: Decoupled the technical architecture into two separate scripts. Created a dedicated spawnPlane script to exclusively manage physics instantiation (Instantiate()), and a secondary modular prefab script to manage audio variables, volume control, and a 5-second maximum clip timer.

UX Architecture & User Research

Designing for Virtual Reality demanded moving away from flat interfaces. The target user group required low cognitive load interfaces that mirrored real-world physical intuitions.

1. Spatial Ergonomics

Calculated optimal user interaction zones based on natural arm-reach radiuses. We placed critical structural UI panels within a 30 to 45-degree comfortable viewing cone to limit neck fatigue.

2. Affordance & Feedback Loops

Because spatial interactions lack physical textures, we mapped multi-sensory feedback loops. Interactive objects change shade on hand proximity, play audio on touch, and trigger precise controller haptics upon click execution.

3. User Testing Insights

Observational telemetry data showed that initial testers tried to swipe virtual items rather than grab them. We altered item geometry to include pronounced handles, decreasing task completion times by nearly 40%.

Reflection & Further Development

An honest evaluation reveals the technical hurdles and design limits of the current prototype alongside a roadmap to fully realize our project aims.

Aim 01

Zero-Friction Immersion

The Reality Check: While eliminating 2D HUD menus kept the user visually immersed, relying purely on physics-based spatial interfaces introduced unintended physical friction; without subtle tactile resistance, menus feel "floaty" and less predictable.

Further Development: Implement variable haptic feedback loops mapped directly to physics actions. Generating a distinct "snap" sensation in the controllers when a player's hand intersects a spatial UI element provides the missing tactile confirmation required for reliable interaction.

Aim 02

Reduce Cognitive Load

The Reality Check: User testing proved that reducing cognitive load goes beyond omitting tutorials. First-time VR users still faced a split-second of confusion trying to swipe certain items before realizing they needed to grab them.

Further Development: Perform a systematic overhaul of asset visual affordances across the entire sandbox, using highly pronounced handles and contextual shader glows that instantly signal how an object is manipulated to drive task-completion speeds even higher.

Aim 03

Feedback & Participation

The Reality Check: The sandbox successfully encourages standard real-life actions (like watering plants or waving sparklers), but the loop feels static because the environment doesn't meaningfully react back to the user over time.

Further Development: Introduce persistent environmental states. Watering plants should trigger an active script collider that dynamically scales up the asset's geometry over time, turning temporary particle loops into living feedback systems.
